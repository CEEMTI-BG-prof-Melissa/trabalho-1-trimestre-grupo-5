Biblioteca Digital Escolar
Funcionais Não Funcionais Inverso

Requisitos Funcionais

  1. O sistema deve recomendar leituras personalizadas para cada turma e acompanhar o desempenho dos alunos em relação à leitura.
  2. O sistema deve registrar sugestões de livros e ter acesso ao relatório sobre quais matérias estão sendo utilizadas.
  3. O sistema deve ter um painel para monitorar o uso da biblioteca de forma eficiente, verificando os alunos que mais acessam o sistema e a frequência de visualização da plataforma.
  4. O sistema deve proporcionar recomendações diretamente da plataforma antes de um exemplar físico.
  5. O sistema deve ter indicações de leitura personalizadas.
  6. O sistema deve permitir que os alunos solicitem o empréstimo de livros disponíveis na biblioteca.
  7. O sistema de gerenciamento do acervo físico e digita.
  8. O sistema de reserva antecipada de exemplares.
  9. O sistema deve notificações automáticas para lembrar da devolução dos livros.
  10. O sitema deve fazer um recurso para anexar atividades complementares aos livros.
  11. O sistema deve ter um ranking com os livros mais lidos na escola.

Requisitos Não funcionais

  1. O sistema deve conter interface simples, acessível e intuitiva para alunos e professores.
  2. O sistema deve conter compatibilidade com celulares, tablets e computadores.
  3. O sistema deve conter alta disponibilidade (funcionar a qualquer momento).
  4. O sistema deve conter desempenho rápido e eficiente ao buscar ou acessar livros.
  5. O sistema deve conter segurança de acesso, restringindo o uso a alunos e professores.
  6. O sistema deve conter privacidade dos dados dos usuários (leituras, acessos, relatórios).

Requisitos Inversos

  1. O sistema não deve impedir os alunos de pegar livros emprestados, a menos que haja uma restrição válida (como multas pendentes ou limite de empréstimos atingido).
  2. O sistema não deve permitir que os livros permaneçam emprestados por um período excessivo sem que haja um aviso ou ação corretiva.
  3. O sistema não deve permitir o acesso de usuários que não sejam alunos ou professores da escola, bloqueando tentativas de login com credenciais inválidas ou de fontes externas.
  4. O sistema não deve ter falta de controle de empréstimos, que possa causar extravios.
  5. O sistema não deve ter ausência de lembretes automáticos, que leve a atrasos frequentes.
  6. O sistema não deve ter dependência exclusiva de livros físicos, dificultando o acesso.
  7. O sistema não deve ter interface confusa ou complicada, especialmente para dispositivos móveis.
